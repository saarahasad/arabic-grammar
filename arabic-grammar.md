# Arabic Grammar Website

## Project Overview

A web-based learning platform for Arabic grammar.

- **GitHub Pages**: Static hosting from the **`main`** branch via **`.github/workflows/deploy-pages.yml`**. In the repo on GitHub: **Settings → Pages → Build and deployment → Source: GitHub Actions**, then merge/push **`main`** so the workflow runs. Site URL: **`https://saarahasad.github.io/arabic-grammar/`**. **`.nojekyll`** turns off Jekyll so files under **`lessons/`** (e.g. **`_template.html`**) are not dropped.
- **Home (`index.html`)**: Course-style study path — units and **every sub-topic** as lesson rows. List items may still use **`href="lesson.html?rule=RULE_KEY"`**; **`course-home-links.js`** rewrites them to **`lessons/<RULE_KEY>.html`** (same as **`getLessonUrl`** in `course-outline.js`). Add outline rows in **`course-outline.js`**, then run **`node scripts/generate-lesson-pages.js`** to create or refresh **`lessons/<rule>.html`**. Rule keys match **`RULES`** in `app.js`. **Arabic labels**: each unit may set **`unitLabelAr`** in **`course-outline.js`** (also shown in the left nav and lesson sidebar). Lesson Arabic titles come from **`RULES[rule].arabicTitle`** in **`app.js`**; **`course-home-arabic.js`** injects them under each English row on the home page (loads after **`app.js`**, which sets **`window.RULES`**). **`course-outline.js`** also exposes **`window.RULE_ARABIC_TITLE`** (same strings as **`RULES.arabicTitle`**) and **`findLessonInOutline(...).titleAr`** so pages that **do not** load **`app.js`** (e.g. Qurʿān Iʿrāb) can still show **English — Arabic** lesson names.
- **Iʿrāb quiz (`iraab-quiz.html`)**: Interactive **15-question** practice quiz (mixed surahs) generated by **`iraab-quiz-generator.js`** from curated **`quran-iraab-data.js`** (via **`buildIraabQuizDatasetFromCurated`** in **`iraab-quiz-page.js`**). **New quiz** redraws; optional **seed** repeats the same question set; **Check answers** scores 15/15.
- **Qurʾān Iʿrāb (`quran-iraab.html`)**: Toolbar **Progress → Mark surah studied** toggles whether you have finished studying the **currently open surah** (saved in **`localStorage`** as **`quranIraabStudiedSurahs`**); tap again to undo. Studied surahs show a **✓** in the Browse sidebar. Covers **every āyah from surah 95 through 114** (At-Tīn through An-Nās), plus **optional curated āyāt outside that range** when listed in **`quran-iraab-data.js`** with full **`ayahText`** and **`rows`** (merged and sorted into the same viewer — e.g. **al-Baqarah 2:17**). **`quran-text-96-114.js`** holds full Uthmani text per āyah for **95–114**, including **95** merged from the same API (regenerate with **`scripts/fetch-quran-text-96-114.py`** or merge surah 95 as needed). **`quran-iraab.js`** merges that with curated **`window.QURAN_IRAAB_AYAH`** in **`quran-iraab-data.js`**: **two menus** — Arabic **surah** names with **surah number** (Eastern Arabic digits) before each name, then **āyah** numbers (Eastern Arabic digits). Verse card headings show **surah no. — name — āyah no.** **View**: selecting a surah shows **all of its āyāt in one scrollable column**; the **āyah** menu (and **←** / **→** keys when not in a field) move **within that surah** and scroll the chosen verse into view (without re-rendering the whole surah when only the āyah changes). Where no curated **`rows`** exist, the page shows **Iʿrāb is not available for this ayah** (EN + AR). Surah **101** (Al-Qāriʿah) is curated per āyah (**`101-1` … `101-11`**). (Surah 103 is curated per āyah: **`103-1` … `103-3`**.) Curated Arabic **`analysisParts`** for **103** (all āyāt), **104** (Āyāt 1–9), **105**, **108**, **110**, and **112** were aligned with user-pasted madrasa-style iʿrāb (including **الْهُمَزَة** per supplied verse-by-verse paste). **Reading mode** (toolbar area): **Beginner** vs **Scholar** — persisted as `localStorage` **`quranIraabGrammarMode`** (`'beginner'` default, `'scholar'` for colour key, literal gloss line under each word’s Arabic analysis, and a fixed **`#qg-tooltip`** on grammar terms with `data-term` / `data-roman` / `data-def`). **Beginner** shows class-based term colours (`.irab-term.t-clause` … `.t-verb`) from **`colorizeIraabClasses`** in **`iraab-colorize.js`**; each analysed word is a **card** (Amiri Arabic panel + English panel: Lora lead sentence + DM Sans full `analysisEn`, pill from first lesson link). **UI**: **`quran-iraab.css`** — off-white page base (`#FAFAF8`), **Browse** + **toolbar** as white cards; **Cairo** for Arabic toolbar/chrome labels (e.g. السُّورَةُ / الْآيَةُ); body/DM Sans + **Amiri** for Qurʾānic text and iʿrāb; single-column intro instructions; **Show prompts** + **Tashkīl** outlined (shared control height ~44px, radius 10px). Left **Browse** pane lists **surahs only** (one row per surah; tap opens that surah). The **āyah** dropdown in the toolbar moves within the surah. Narrow screens use **Surahs** to open the same pane over a backdrop. Verse block, **Breakdown** when analysis exists — each **word** is a card; each card may show a **think-first prompt**; **clicking the prompt** replaces it with the full Iʿrāb (not an accordion). **Show prompts** (toolbar checkbox; persists as `localStorage` `quranIraabShowPrompts` = `'1'`) turns on think-first prompts; **default** is direct Iʿrāb only (no prompts). Legacy `quranIraabDirectIraab` is migrated once. Optional **`prompt`** / **`promptEn`** per row in data; per-verse **Reveal all** / **Show prompts again** when prompts are on; **Scholar** mode: **lesson rule links** in the analysis stay **underlined** with **base `color`** from **`ruleLinkColorForRule(rule)`** in **`iraab-colorize.js`**; **Beginner** word cards **omit** rule-link underlines so **`.irab-term`** pill backgrounds show without dotted lines. **`colorizeIraab(text)`** (longest-match, diacritic-insensitive) colors **both** `text` and `link` segments in Scholar mode before tooltip annotation; **`prompt`** / **`promptEn`** and **`ayahText`** stay plain. **Harakah (تشكيل)**: **`utils/harakah.js`** (and **`utils/harakah.ts`**) — **`stripHarakah`**, **`hasHarakah`**, **`readShowHarakah`**, **`writeShowHarakah`**, **`displayArabic`**. Preference in **`localStorage`** as **`quran-iraab-show-harakah`** (default: show). Toolbar strips diacritics from iʿrāb analysis, prompts, and nav/surah labels — **not** from **`ayahText`**, **`row.segment`**, or the **Phrase** column header **الْمُعَرَّفُ**. Links go to **`lessons/<rule>.html`** with **bilingual** `title` / `aria-label` (**outline English — Arabic**); the duplicate Arabic lesson title is **not** shown under each link (only the linked grammatical label from **`analysisParts`**). **Course home** (`home.css`): lesson Arabic sub-labels and sidebar unit Arabic use stronger contrast (teal) for readability. **Per-word **Explanation** (textarea under each word)**: when saved, text shows as part of the card with an **Edit** control; use **Edit** to change. Saved text is in **`localStorage`** as **`quranIraabWordNotes`** (keys **`95-1::0`** = āyah id + word row index). Toolbar **Download notes** saves **`quran-iraab-notes-YYYY-MM-DD.json`** (`format: quran-iraab-notes-v2`, `notes`, sorted **`words`** with `segment` and surah/āyah metadata). Curated examples include **الْإِخْلَاص** (112:1–4), **الْفَلَق** (113:1–5 — **113:3–5**: **`وَمِن شَرِّ`** as مَعْطُوف on **`مِن شَرِّ`** from 113:2; **`إِذَا`** as ظَرْفُ زَمَانٍ + **مَفْعُولٌ فِيهِ** لِ**أَعُوذُ** with the following مَاضٍ in مَحَلُّ جَرٍّ بِالْإِضَافَةِ; **113:4** **`فِى ٱلْعُقَدِ`** متَعَلِّقٌ بِ**النَّفَّاثَاتِ**), **النَّاس** (114:1–6), **الْكَافِرُون** (109:1–6), **الْكَوْثَر** (108:1–3), **النَّصْر** (110:1–3), **الْمَسَد** (111:1–5), **قُرَيْش** (106:1–4), **الْمَاعُون** (107:1–7), **الْفِيل** (105:1–5), **الْهُمَزَة** (104:1–9), **الزَّلْزَلَة** (99:…), **الْعَصْر** (103:1–3), **الْعَادِيَات** (100:1–11 — per-āyah **`rows`** match user-supplied madrasa-style Arabic analysis), **الْقَارِعَة** (101:1–11 — per-āyah **`rows`** match user-supplied madrasa-style Arabic analysis), **التَّكَاثُر** (102:1–8 — per-āyah **`rows`** use the pasted madrasa-source Arabic verbatim), **الْعَلَق** (96:1–19), **التِّين** (95:1–8), **الْقَدْر** (97:1–5), etc. **`analysisParts`**: `[{ type: 'text', text }, { type: 'link', text, rule }]`. **`SHOW_QURAN_IRAAB_ENGLISH`** in **`quran-iraab.js`** is **`false`**: English **`translationEn`**, **`analysisEn`** (and **`utils/iraab-simple-en.js`** fallback), **`promptEn`**, beginner **→** glosses, scholar literal gloss, and grammar-term **`data-def`** tooltips are omitted from the DOM; **`quran-iraab-data.js`** is unchanged (set the flag to **`true`** to show that English again).
- **`lesson.html?rule=…` (legacy / fallback)**: **`lesson-ui.js`** **redirects** to **`lessons/<rule>.html`** when that rule is in the course outline. If you stay on **`lesson.html`** (e.g. rule not in outline), the main column is filled by **`fetch`** from **`lessons/<rule>.html`** via **`lesson-content.js`** (same HTML as the map panel).
- **Standalone lesson pages (`lessons/<rule-id>.html`)**: **Source of truth** for lesson text (migrated from the old `RULES.content` in `app.js`). Each file includes the **reading column** inside **`#lesson-main-inner`** (title, Arabic title line, **`rule-body`**). **`app.js`** keeps only **`RULES`** **metadata** (`title`, optional `arabicTitle`) for the mind map and titles. **`lesson-content.js`** loads **`#lesson-main-inner`** HTML via **`fetch`** for the **map panel** (right side) and for **`lesson-ui.js`** when the main column is empty or **`data-hydrate-from-rules`** is set. Edit lesson HTML directly to add **images** / **exercises**. Regenerate from a backup **`app.js`** using **`scripts/migrate-app-rules-to-lesson-files.js`** only if you need to bulk-rebuild bodies. Asset paths from `lessons/` use **`../assets/…`**. Helpers: **`.lesson-figure`**, **`.lesson-exercise`** in **`lesson-khan.css`**.
- **Map of Arabic Grammar (`map.html`)**: Optional **mind map**; clicking a rule **fetches** the same **`lessons/<rule>.html`** fragment used on standalone lesson pages and shows it in the **right-hand panel** (with reading rail). **`lesson-content.js`** must load **before** **`app.js`**. Use a local HTTP server (not `file://`) so **`fetch`** works.

## Architecture

- **Arabic typography (site-wide)**: Google Fonts **`Amiri`** for Qurʾānic text, iʿrāb analysis, lesson examples, and other classical Arabic; **`Cairo`** for Arabic UI (nav, mind-map node labels, course home Arabic, lesson sidebar Arabic, Qurʾān toolbar labels). Loaded via **`@import`** at the top of **`styles.css`** and utility classes **`.arabic-classical`** / **`.arabic-ui`**; **`--font-arabic`** / **`--font-arabic-ui`** on **`:root`**. The legacy local **KFGQPC Uthmanic** face is no longer used for Arabic display.
- **Frontend**: Vanilla HTML, CSS, JavaScript. **`iraab-colorize.js`** exposes **`colorizeIraab(text)`** (longest-match, diacritic-insensitive) and **`ruleLinkColorForRule(ruleId)`** for lesson-link hues on the Qurʾān viewer. **`utils/harakah.js`** (see **`utils/harakah.ts`**) exposes **`stripHarakah`**, **`readShowHarakah`**, **`writeShowHarakah`**, **`displayArabic`** for the Qurʾān viewer harakah toggle. **Qurʾān Iʿrāb** (`quran-iraab.html`) uses keyword coloring on analysis text (no gray chip backgrounds on rows). Optional **`iraab-visual/`** (React + Vite + Tailwind) exists for component experiments; build goes to **`iraab-visual/dist/`** and is **not** required for the main site.
- **Structure**:
  - **Home**: `index.html` + `home.css` + `home.js` (sidebar scroll highlight). Global chrome: `styles.css` (`.site-header` / `.site-nav` shared with the map page).
  - **Map page**: `map.html` loads **`lesson-content.js`** then **`app.js`** (mind map + panel content from **`lessons/*.html`**). **Large screens**: Mind map and lesson panel share the viewport. **Small screens**: Full-width slide-in lesson over a light overlay.

### Educational card component (`edu-card`)

- **File**: `components/edu-card.js` — registers the custom element `<edu-card>` (reusable “Card” in plain HTML; no React).
- **Styling**: Encapsulated in Shadow DOM — soft slate gradient (`145deg`, `#f1f5f9` → `#e2e8f0`), `16px` radius, layered shadow, inner top highlight, divider + “LEARN MORE ▼” footer (`12px`, `#94a3b8`, `letter-spacing: 0.5px`). Hover: `translateY(-4px)` and deeper shadow; `transition: 0.25s ease`.
- **Typography**: Arabic title `#1e293b` (uses `--font-arabic` when set on `:root`); English subtitle `#64748b`, `margin-top: 4px`.
- **Layout**: `min-width: 180px`, centered text, `padding: 16px 20px` on the card surface.
- **Slots**: `arabic`, `english`, optional `learn-more` (default text `LEARN MORE ▼`).
- **Optional**: `interactive` attribute — `role="button"`, keyboard (Enter/Space) activates a `click` on the host for wiring in JS.
- **Load**: `<script src="components/edu-card.js" defer></script>` before `app.js` in `map.html` (not used on the course home page unless you embed `<edu-card>` there).

### Mind map UI (design system)

- **Typography**: Latin UI uses **Inter** (Google Fonts). Short Arabic on map nodes and chrome uses **Cairo**; Qurʾānic and lesson-body Arabic uses **Amiri** (see site-wide Arabic typography above).
- **Surface**: Nodes are **clean white** (`.node` / `.card-surface`) with a soft border and tiered **category accent** on the **top edge** only: hub **indigo** (`--accent-hub`), Kalimaat **slate**, Noun **emerald**, Verb **amber**, Particle **violet**, Sentence **teal**, Iʿrāb **rose** (`styles.css` `:root`). Section headers use a **4px** stripe; main grid nodes **3px**; deeper sub-grid / raf signs **2px**. Hover lift and shadow unchanged; **selected** uses an **indigo outer ring** (box-shadow) so the colored stripe is not covered.
- **Lesson panel**: **White** (`#ffffff`) with a **4px left border** in the same accent as the active topic (no full-panel pastel gradients).
- **Background**: Flat **off-white paper** `#FAFAF7` (`--canvas-base`). Optional **barely-there grid**: neutral lines at **~3% opacity**, **40px** spacing (`--canvas-grid-line`, `--canvas-grid-size`). **No** tinted gradients. For a completely plain page, add class **`paper-plain`** on `<html>` (removes grid). **Nodes**: soft material elevation `0 2px 6px rgba(0,0,0,0.05)` (`--card-shadow`).
- **Connectors**: SVG paths class `connection-line`, stroke `#b8bcb5`, width `1.5` (see `drawConnections` in `app.js`). The SVG uses `scrollWidth` / `scrollHeight` so lines are not clipped; **Grammar Rules → Kalimaat / Sentence / Iʿrāb** use **bottom → top** anchors with a vertical Bézier; lines redraw on scroll, resize, and `ResizeObserver`. Collapsed Kalimaat child row uses **zero width** so it does not stretch the Kalimaat column and misalign connectors.
- **Card chrome**: Section headers may include `data-card-kind` and emoji via `.node-card-icon`; expandable nodes use a top border on **Learn More** as a divider.

## Features

1. **Course home page (`index.html`)**: Ordered units with lesson links to the map; add rows anytime (see HTML comment in `index.html`).
2. **Mind Map Page (`map.html`)**: MindMup-style mind map — clean, minimal, distraction-free. **Hierarchy**: Grammar Rules (center) → Kalimaat (Words), Sentence, I'rāb. **Kalimaat** is the parent of Noun, Verb, Particle — no direct connections from Grammar Rules to these; only Grammar Rules → Kalimaat → (Noun, Verb, Particle). **Sentence** → Nominal / Verbal / **Exception** (`istithna` — الْإِسْتِثْنَاء with إِلَّا); **Nominal** (Learn more) → **إِنَّ وَ أَخَوَاتُهَا** (`inna-sisters` — اسم إِنَّ manṣūb, خَبَر إِنَّ marfūʿ) and **كَانَ وَ أَخَوَاتُهَا** (`kaana-sisters` — الأفعال الناقصة; اسم كَانَ marfūʿ, خَبَر كَانَ manṣūb); **Verbal** (Learn more) → **Faʿil** (`fael`), **Mafʿūl** (`mafool`), and **الْفِعْلُ الْمَجْهُول** (`verb-passive-overview` — passive overview; links to **`verb-past-passive`** / **`verb-present-passive`** for full tables); **Particle** (Learn more) includes **الْمُنَادَى** (`munada` — vocative / حُرُوفُ النِّدَاء). **Interaction rules** (separate): **Node click** → Show info panel. **Learn more button** → Expand or collapse children. Use `event.stopPropagation()` on Learn more so it does not trigger node click. State: `expandedNodes = { kalimaat, noun, verb, verbPastExpand, verbPresentNegation, verbImperative, harf, rafSigns, verbalSentence, nominalSentence }`, `selectedNode = null`. **Expand/collapse**: By default only Kalimaat is shown; click "Learn more" to expand and reveal Noun, Verb, Particle in one horizontal row. Each child expands independently. Arrow icons: ▶ collapsed, ▼ expanded. Selected node: stronger border, slight shadow. Smooth fade + slide animation (200–300ms). Connector lines, color-coded sections, horizontal scroll.
3. **Explanation Panel (“reading workspace”)**: On viewports **≥ 900px**, opening a rule adds `lesson-active` on `.app`: the mind map and lesson sit **side by side** (split view)—no dimmed modal; the overlay is disabled so the map stays fully visible and scrollable. On smaller screens, the panel still **slides in full width** over a **light frosted** overlay (warm paper tint + slight blur), not a dark modal scrim. **Chrome**: sticky `panel-reading-rail` (Lesson kicker + short hint); close control is **Back to map** on desktop and **×** on narrow layouts. Close: button, overlay tap (where overlay is shown), or **Escape**. **Textbook layout (all rule bodies)** — `openPanel` wraps content in `panel-reading-main` and injects `<div class="rule-body textbook-lesson" data-rule="…" data-topic="…">` around every `RULES[…].content`. **Topics** (for paper gradient + Arabic title underline tint): `topic` (intro, kalimaat, words, coming-soon), `noun` (word-level / Kalimaat subtree except particles & verbs), `verb`, `harf` (incl. munada), `sentence` (sentence structure, nominal/verbal branches, faʿil/mafʿūl, passive overview, istithnāʾ), `irab`. **Shared styling**: reading measure (`max-width: 42rem`), serif `h2`, section `h3` with light rule, calmer `p.definition`, softer cards/tables (`note-box`, `example-row`, `quran-example`, comparison & pronoun tables). **Noun-only body**: structured lesson remains inside `data-rule="noun"` (`lesson-sheet`, marker grid). Implementation: `getRuleTopic` + `escapeHtml` in `app.js`; `styles.css` blocks “Explanation panel” and “Textbook-style explanation panel”.

4. **Parts of Speech (كَلِمَات)** — three divisions:
   - **Noun (الْإِسْم)**: Pronouns, Definite & Indefinite, **Special nouns** (`special-nouns` — demonstratives, relatives, interrogatives; **Magnified nouns / five special nouns** `magnified-nouns` — الْأَسْمَاءُ الْمُكَبَّرَة · الْأَسْمَاءُ الْخَمْسَة (أَبٌ، أَخٌ، ذُو، حَمٌ، فَمٌ as مُضَاف; حَم/فَم muḍāf shapes not in Qurʾān; ذُو/ذَات), **المبني والمعرب** (`mabni-muarab` — unchangeable vs changeable nouns; مُنْصَرِف vs غَيْرُ مُنْصَرِف; textbook examples), **الممنوع من الصرف** (`mamnu-sarf` — diptote overview: مَبْنِيّ / مُنْصَرِف / مَمْنُوعٌ مِنَ الصَّرْف, categories & examples), **verbal noun** (`masdar` — الْمَصْدَر / Lesson 26: primitive vs derived; فَتْحٌ/سَمْعٌ; Qurʾānic examples 31:11, 17:31, 66:8, 92:4, 48:1, 2:217, 3:191, 41:22)
   - **Verb (الْفِعْل)**: Past (الْمَاضِي) — **Learn more** reveals **مَاضِي مَجْهُول** (`verb-past-passive`), **فَعْلٌ مُضَاعَفٌ** / geminate verbs (`verb-mudaaf`: second and third root letter the same, e.g. `مَدَّ`, mudāriʿ with `لَمْ`/`لَنْ`, imperative/prohibition samples, and a **full vocalized vocabulary table** of mudāriʿ + māḍī forms from the textbook), **فِعْلٌ مَهْمُوزٌ** (`verb-mahmuz`: hamza in the root — three sample charts for أَخَذَ / سَأَلَ / قَرَأَ, Quranic examples, and **vocabulary** by first/second/third hamza), and **الْفِعْلُ الْأَجْوَفُ الْوَاوِيُ** (`verb-ajwaf-wawi`: hollow verbs with middle **wāw** — `قَالَ` past / present / imperative tables, summary rules, Quranic examples, and **vocabulary** mudāriʿ + māḍī from the textbook); **الْأَجْوَفُ الْيَائِيُ** (`verb-ajwaf-yaee`: middle **yāʾ** — `بَاعَ` tables, rules, Quranic samples, vocabulary); **أَجْوَفٌ — اسْتِثْنَاء** (`verb-ajwaf-exception`: verbs like `خَافَ` / `يَخَافُ` where the middle **alif** stays in the mudāriʿ, exception vocabulary كَادَ/نَالَ/نَامَ/مَاتَ/زَالَ/شَاءَ, examples); **وَاوُ الْأَوَّلِ** (`verb-initial-waw`: assimilated first **wāw** — `وَعَدَ`/`يَعِدُ` conjugations, Quranic samples, vocabulary for initial-و verbs); **الْفِعْلُ النَّاقِصُ الْوَاوِيُ** (`verb-naqis-wawi`: defective last **wāw** — `دَعَا` / `يَدْعُو` past / present / imperative tables, rules, Quranic examples, vocabulary including نَصَا/نَفَا/صَفَا/ضَحَا pairs). Present/Future (الْمُضَارِع) — **Learn more** reveals **مُضَارِع مَجْهُول** (`verb-present-passive`), **لَامُ التَّوْكِيدِ** / double emphasis (`verb-present-double-emphasis`), plus negation / conditional / jussive / accusative sub-nodes; Imperative (الْأَمْرُ) — panels: tri-literal / فَعَلَ intro; past — middle-letter types, full conjugation table, subject-after-verb (masc. & fem.), Quranic samples, قَدْ/لَقَدْ/مَا, تُمُو before pronoun; present — يَفْعَلُ table, four prefixes, suffixes, rafʿ note, six past→present patterns, سَ/سَوْفَ, examples; under Present (Learn more): **Negation (نَفْيٌ وَ نَهْيٌ)** — lists لَنْ، وَلَمْ، لَا، مَا، لَيْسَ as negation words (with pointers to jussive/naṣb sections); prohibitive لَا + jussive mudāriʿ conjugation table (نَفْيُ الْفِعْلِ), contrast with negating لَا and مَا (no change to verb); **لَيْسَ** (فِعْلٌ نَاقِصٌ) full conjugation table and Quranic samples; **Conditional words (حُرُوفُ الشَّرْط)** — own mind map node; panel body `CONDITIONAL_HURUF_SHART_CONTENT` in `app.js` (vocabulary, شَرْط / جَوَابُ الشَّرْط, إِنْ examples, past verb with present sense, فَ / لَوْ+لَ / إِلَّا); **Jussive particles (حُرُوفٌ جَازِمَة)** — overview (لِ، لَا، لَمْ، لَمَّا، إِنْ), note linking to Conditional words; لَمْ، لَمَّا, hidden إِنْ; **Accusative particles (حُرُوفٌ نَاصِبَة)** — note that conditionals use jazm, not naṣb; mudāriʿ in naṣb after لَنْ، حَتَّىٰ، أَنْ/لِأَنْ/أَلَّا، كَيْ/لِكَيْ/لِكَيْلَا، لِ (لَامُ التَّعْلِيلِ); same detailed HTML as Iʿrāb → Naṣb (`ACCUSATIVE_PARTICLES_MUDARI_CONTENT` in `app.js`); imperative — فِعْلُ الْأَمْر: 2nd-person table (masc./fem. × sing./dual/plural), three-step formation from mudāriʿ, hamzat al-waṣl and vowel rules, Quranic samples; child node **Lam al-amr (لِـ)** — لَامُ الْأَمْر for 1st/3rd person (لِيَفْعَلْ, لِتَفْعَلْ, …), jussive after لِـ, وَلْـ/فَلْـ after وَ/فَ, examples. **Stem color**: In verb tables and many examples, the tri-literal stem (ف–ع–ل) uses class `verb-root` (amber-brown text); prefixes and suffixes use `verb-affix` (muted gray).
   - **Particle (الْحَرْف)**: Letters (حُرُوفُ الْهِجَاء), Particles with meaning (حُرُوفُ الْمَعَانِي), Prepositions (حُرُوفُ الْجَرِّ), **Vocative** (`munada` — الْمُنَادَى, يَا / يَا أَيُّهَا / يَا أَيَّتُهَا, marfūʿ vs manṣūb)

5. **Pronouns (ضَمَائِر)** — under Noun, hierarchical branch:
   - Detached (مُنْفَصِلَة): stand alone
   - Attached (مُتَّصِلَة): with nouns, verbs, particles
   - With Nouns: possession (كِتَابُهُ, رَبُّكَ)
   - With Verbs: object (خَلَقَهُ, خَلَقَنِي)

6. **I'rāb (الْإِعْرَاب)** — panel content includes reference tables: (1) the four types with original vowel sign and term (مَرْفُوع، مَنْصُوب، مَجْرُور، مَجْزُوم) and examples؛ (2) where each type appears (nouns vs. present verb); (3) note that particles and past/imperative verbs are مَبْنِيّ. Standalone **`lessons/irab.html`** adds a **Linked lessons** table (English + Arabic + `lessons/<rule>.html`) to **mabni-muarab**, sentence topics, **fael**/**mafool**, **harf-jarr**, jussive/accusative particles, etc. **`lessons/mabni-muarab.html`** includes section **C** on **مَبْنِيّ** in analysis: **مَبْنِيّ عَلَى …** (fixed vowel) vs **مَبْنِيّ فِي مَحَلِّ …** (fixed form, syntactic slot). Mind map: **Raf'** uses Learn more to reveal four child nodes (Dammah, Wāw, Alif, Nūn). Panel content: articulated i'rāb (original vs. secondary signs); four signs of rafʿ (ضمة، واو، ألف، نون); when dammah applies (singular, broken plural, sound fem. plural, bare mudāriʿ); sections for جمع التكسير، الجمع المؤنث/المذكر السالم، الأفعال الخمسة (table ثبوت النون), المثنى, الأسماء الخمسة when muḍāf. **Naṣb** panel: noun-focused intro plus shared block `ACCUSATIVE_PARTICLES_MUDARI_CONTENT` (accusative particles table; signs on the mudāriʿ; لِ + فَعَلَ table; لَامُ الْأَمْر vs لَامُ التَّعْلِيلِ; Quranic examples) — duplicated under **Verb → Present → Accusative particles**.

7. **Demonstratives (أَسْمَاءُ الْإِشَارَة)** — panel: near vs. far; tables (sing./dual/plural, masc./fem.); dual rafʿ (ـانِ) vs naṣb/jarr (ـيْنِ); far forms ending in كَ; Quranic examples; special uses of ذَلِكَ/تِلْكَ for emphasis and تِلْكَ with some plurals.

8. **Definite & Indefinite Nouns (نَكِرَةٌ & مَعْرِفَةٌ)**:
   - Indefinite (نَكِرَةٌ): unspecific nouns, marked by تَنْوِين (tanween)
   - Definite (مَعْرِفَةٌ): specific nouns, marked by ال (Al-)
   - Tanween types: Bait-un (ٌ), Bait-an (ًا), Bait-in (ٍ)
   - Rule: ال and تَنْوِين cannot coexist
   - Exception: proper names like مُحَمَّدٌ

## Database Schema

No database - static content for now. **`app.js`** holds **`RULES`** metadata (`title`, `arabicTitle`) and assigns **`window.RULES`** for scripts that only load metadata (e.g. **`course-home-arabic.js`** on the home page). Lesson HTML bodies live under **`lessons/`**.

### iʿrāb quiz generator (`iraab-quiz-generator.js`)

- **`generateIraabQuiz(dataset, options?)`**: builds **15** randomized questions in **Arabic**, mixed surahs (one word per surah, **15 distinct surahs**), **no duplicate words** across the selected pool, **4 options** per item, **`answer`** always matches one option. **`options.seed`** (number) makes runs reproducible; default uses `Date.now()`.
- **Input `dataset` rows** (array of objects):

| Field | Purpose |
|--------|---------|
| `surah` | Surah number (required). |
| `ayah` | Āyah number or label (required for grouping). |
| `word` | The segment being quizzed (required). |
| `type` | `اسم` / `فعل` / `حرف` (or common synonyms). |
| `i3rab` | Full analysis string (Arabic). |
| `role` | Syntactic role (e.g. فاعل، مفعول به). |
| `extra` | Notes (e.g. ضمير مستتر تقديره «…»). |
| `ayahText` | Optional full āyah text for stems **ayah_count** / **multi_select_verbs**. |

- **Output**: `[{ question, options, answer, type }, …]` with **`type`** an **Arabic** label for the template (see `IRAAB_QUIZ_QUESTION_TYPE_AR` / `QUESTION_TYPE_AR` in the script). Internal keys (`word_type`, …) are listed in `IRAAB_QUIZ_QUESTION_TYPES`. The **12** templates fill **15** slots with **some types repeated**; the final list is **shuffled**.
- **Requirements**: dataset must cover **at least 15 different surahs** with enough unique `word` values to pick one per surah.

### Qurʾān Iʿrāb data (`quran-iraab-data.js`)

| Field | Purpose |
|--------|---------|
| `id` | Stable string key for URL hash and `<select>` (e.g. `109-2`). |
| `surah` | Surah number (number). |
| `surahNameAr` | Arabic surah name for the header. |
| `ayah` | Āyah label (number or string e.g. `١–٢`). |
| `ayahText` | Full āyah text (Uthmani-style as in your source). |
| `translationEn` | Optional English gloss. |
| `rows` | Array of `{ segment, analysisParts }`. |
| `iraabUnavailable` | Optional; if `true` or `rows` empty after merge, UI shows unavailable (usually omit and rely on merge). |
| `segment` | The word/phrase being analysed (shown in blue). |
| `analysisParts` | Ordered segments: `{ type: 'text', text }` or `{ type: 'link', text, rule }` where **`rule`** is a lesson key present in **`course-outline.js`**. |
| `prompt` | Optional Arabic “think first” question shown **before** the full Iʿrāb when toolbar **Show prompts** is on (default off = direct Iʿrāb). |
| `promptEn` | Optional English hint with the prompt. If **`prompt`** is omitted, **`quran-iraab.js`** shows a short **Before revealing —** line and **clickable lesson titles** from **`analysisParts` links** (or a one-line “which sign?” hint when there are no links). |
| `noPrompt` | If `true`, that row always shows the full analysis (even when **Show prompts** is on). |
| `analysisEn` | Optional English explanation of the same iʿrāb as **`analysisParts`** (shown on the English side of each word card in **`quran-iraab.html`**). If omitted, **`utils/iraab-simple-en.js`** builds a simple English line from **`analysisParts`** (rule labels + phrase translation of common Arabic iʿrāb strings). Batch-generate curated text with **`scripts/fill-quran-iraab-analysis-en.mjs`** and **`ANTHROPIC_API_KEY`** (Claude), or put the key in **`.env`** at the repo root as `ANTHROPIC_API_KEY=…`. |

Add new migrations (new āyāt) in the **same file** by pushing to **`QURAN_IRAAB_AYAH`**. **`quran-text-96-114.js`** is the canonical source for āyah text in the **95–114** viewer; **`quran-iraab-data.js`** only needs curated analysis rows for those surahs. For **āyāt outside 95–114**, supply **`id`** (`surah-ayah`), **`surah`**, **`surahNameAr`**, **`ayah`**, full **`ayahText`**, and **`rows`** — **`quran-iraab.js`** merges them into the same page list. **Surah 95 (At-Tīn)** and **Surah 97 (Al-Qadr)** are filled with **detailed** per-word rows plus sentence-level notes (e.g. 95: قَسْمٌ، جَوَابُ قَسْمٍ، تَعَطُّفٌ، اسْتِئْنَافٌ، لَيْسَ; 97:5 includes جُمْلَةٌ اِسْتِئْنَافِيَّةٌ لَا مَحَلَّ لَهَا مِنَ الْإِعْرَابِ، and **حَتَّىٰ** as جَرٌّ وَغَايَةٌ with **مَطْلَعِ ٱلْفَجْرِ** as جَارٌ وَمَجْرُورٌ **مُتَعَلِّقٌ** بِ**تَنَزَّلُ** from 97:4). **Surah 114 (An-Nās)** follows the same **113-style** opening (جُمْلَةُ «قُلْ» اسْتِئْنَافِيَّةٌ، «أَعُوذُ» وَمَا بَعْدَهَا مَقُولُ الْقَوْلِ)، **بَدَلٌ / عَطْفُ بَيَانٍ** on **114:2–3**, **جَارٌ وَمَجْرُورٌ** phrases tied back to **أَعُوذُ** or **يُوَسْوِسُ** where relevant, **صِلَةُ الْمَوْصُولِ** and alternative محلّ analyses for **الَّذِي** on **114:5**, and **مِنَ الْجِنَّةِ** / **وَالنَّاسِ** on **114:6** (including optional **حَالٌ مَحْذُوفَةٌ**). **Tashkīl on analysis**: link and text spans in **`analysisParts`** are stored fully vocalized (ḥarakāt) so the Qurʾān viewer matches the usual lesson/iʿrāb coloring style; bulk helpers live in **`scripts/vocalize-quran-iraab-data.mjs`** (run from an unvocalized backup if you need to regenerate — then spot-check `عَلَى الضَّمِّ` vs nominative terms and quoted phrases). **English iʿrāb glosses**: optional **`analysisEn`** per row; fill with **`scripts/fill-quran-iraab-analysis-en.mjs`** (Anthropic Claude API; `--dry-run` counts gaps).

- **`kaana-sisters`**: **كَانَ وَ أَخَوَاتُهَا** — incomplete verbs (**الْأَفْعَالُ النَّاقِصَةُ**) before a nominal sentence; **اسْمُ كَانَ** marfūʿ, **خَبَرُ كَانَ** manṣūb; contrast with **`inna-sisters`**; example الْجَوُّ حَارٌ / كَانَ الْجَوُّ حَارًّا; sample verbs **كَانَ، أَصْبَحَ، لَيْسَ**; note on further sisters in fuller grammars.
- **`verbal-sentence`**: Mentions third child **`verb-passive-overview`**.
- **`verb-passive-overview`**: **الْفِعْلُ الْمَجْهُول** — active vs passive; **نَائِبُ الْفَاعِل** marfūʿ; examples شَرِبَ الْوَلَدُ الْمَاءَ / شُرِبَ الْمَاءُ; stem vowels **فُعِلَ** / **يُفْعَلُ**; pointer to **`verb-past-passive`** and **`verb-present-passive`**.
- **`mafool`**: **الْمَفْعُول** — intro linking **نَصْب** to objects of the verb; **five kinds**: **مَفْعُولٌ بِهِ** (direct — e.g. 29:44, 2:251), **مَفْعُولٌ مُطْلَقٌ** (verbal noun emphasis — 17:12, 84:8), **مَفْعُولٌ فِيهِ** (time/place — 54:26, 4:31; lists لَيْلًا/نَهَارًا/… and فَوْقَ/تَحْتَ/…), **مَفْعُولٌ لَهُ** (aim/purpose — 14:28, 4:38), **مَفْعُولٌ مَعَهُ** (with — 10:71, 66:6); then **اسْمُ الْمَفْعُول** passive-participle pattern (existing tables).
- **`munada`**: **الْمُنَادَى** — **حُرُوفُ النِّدَاء**; marfūʿ (proper name, intended common noun, يَا أَيُّهَا/أَيَّتُهَا + الْ); manṣūb (mudāf, unintended common noun); examples يَا زَيْدُ، يَا عَبْدَ اللَّهِ، يَا جَاهِلًا تَعَلَّمْ.
- **`harf-jarr`**: **حُرُوفُ الْجَرِّ** — (1) Core particles: **مِنْ، إِلَى، عَنْ، عَلَى، فِي، بِ، كَ، لِ، حَتَّى، رُبَّ، مُنْذُ/مُذْ، وَ/تَ** (oaths), **خَلَا، عَدَا، حَاشَا**; (2) ẓurūf / iḍāfah-first words with same majrūr effect: **مَعَ، عِنْدَ، لَدُنْ/لَدَى، أَمَامَ، قَبْلَ، بَعْدَ، بَيْنَ، فَوْقَ، تَحْتَ، وَرَاءَ، حَوْلَ، دُونَ، غَيْرَ، مِنْ دُونِ، بَعْضُ، كُلُّ**; notes on nouns vs pronouns; **إِلَى/عَلَى** + suffix (**يْ**); **لِ** → **لَ** before suffixes except **لِي** / **لَيَّ**; Qurʾānic samples (e.g. تَاللَّهِ، وَالْعَصْرِ، وَلِلَّهِ/لِرَسُولِهِ، حَتَّىٰ مَطْلَعِ، كَالْأَنْعَامِ، لَكُمْ وَلِيَ، وَإِنَّ لَكَ لَأَجْرًا، لَهُ مُلْكُ، فَوْقَ/تَحْتَ، مَعَ/عِنْدَ/لَدَى/غَيْرَ).
- **`idafah`**: **الْإِضَافَة** — **مُضَاف** / **مُضَافٌ إِلَيْهِ** rules (no **الْ**/tanwīn on first part; second part always majrūr; **الْ** or tanwīn on second part as applicable); table of **أَصْلِيَّة** vs **فَرْعِيَّة** jarr signs (kasrah vs dual / sound masc. plural / diptote); pronoun suffix as **مُضَافٌ إِلَيْهِ** (e.g. كِتَابُهُ).
- **`tawabi`**: **التوابع** — definition (**مَتْبُوع**); four types (**نَعْت**, **عَطْف**, **تَوْكِيد**, **بَدَل**); pointers to **`naat`** and **`atf`** panels for detail.
- **`naat`**: **النَّعْت** / **صِفَة** — **مَنْعُوت** / **مَوْصُوف**; agreement in case, definiteness, number, gender; adjective-after-noun order; masc./fem. × indefinite/definite table (قَلَمٌ جَدِيدٌ / الْقَلَمُ الْجَدِيدُ / سَيَّارَةٌ جَمِيلَةٌ / السَّيَّارَةُ الْجَمِيلَةُ).
- **`sifah-mawsuf`**: **الصِّفَةُ وَالْمَوْصُوفُ** — descriptive phrase (noun + adjective); textbook-style sections A–D (gender, number, definiteness, iʿrāb) with examples; long list of Qurʾānic citations (same topic as **`naat`**, expanded).
- **`atf`**: **الْعَطْف** — **حَرْفُ الْعَطْف**; **الْمَعْطُوفُ عَلَيْهِ** vs **الْمَعْطُوفُ** (e.g. سَافَرَ أَحْمَدُ وَخَالِدٌ); table of **وَ، فَ، ثُمَّ، أَوْ، أَمْ، إِمَّا، بَلْ، لَا، لَكِنْ، حَتَّىٰ** with glosses; iʿrāb rule (**مَعْطُوف** matches **مَعْطُوفٌ عَلَيْهِ** in rafʿ/naṣb/jarr); examples **رَأَيْتُ زَيْدًا وَعَمْرًا** (manṣūb), **مَرَرْتُ بِزَيْدٍ وَعَمْرٍ** (majrūr).
- **`noun`**: Section **عَلَامَاتُ الْإِسْم** (tanween, الْ, kasrah, tāʾ marbūṭah, after jar, asmāʾ ishārah / ḍamāʾir / mawṣūl); mind map links to **`mamnu-sarf`**. **Panel body**: marker grid + sample + **Continue on the map** — `data-rule="noun"` / `.lesson-sheet` (global textbook chrome: Features §2).
- **`mabni-muarab`**: **الْمَبْنِي وَالْمُعْرَبُ · الْأَسْمَاءُ الْمَبْنِيَّةُ وَالْمُعْرَبَةُ** (Lesson 24 style) — **مَبْنِيّ**: same form in all positions (pronouns, demonstratives, relatives, names like **مُوسَى** / **عِيسَى**); Qurʾānic samples; **مُعْرَب**: (1) **مُنْصَرِف** full triptote endings + tanwīn, **نُوح** examples; (2) **غَيْرُ مُنْصَرِف** — no tanwīn, only **ضَمَّة** / **فَتْحَة** (no **كَسْرَة** on the last letter); **فِرْعَوْن** and listed names; note: noun **إِعْرَاب** is never **سُكُون**.
- **`mamnu-sarf`**: **الْمَمْنُوعُ مِنَ الصَّرْف** — contrast **مَبْنِيّ** / **مُنْصَرِف** / **مَمْنُوعٌ مِنَ الصَّرْف**; no tanwīn / no kasrah on last; categories (foreign names, some prophetic names, ـان names, colours, some plurals, some fem. names, tāʾ marbūṭah masc., angels, places); prophetic exceptions **مُحَمَّدٌ، نُوحٌ، … صَالِحٌ**.
- **`vocab-body-parts`**: **مُفْرَدَاتُ الْجَسَد** — Lesson 16 vocabulary: body parts and related nouns in singular/plural (two tables, textbook pp. 59–60); pairs with **`number-plural`** for plural morphology.
- **`number-plural`**: **جَمْع** — two types: **جَمْعٌ سَالِمٌ** (sound plural: dual **ـَانِ** / **ـَيْنِ**; masc. pl. **ـُوْنَ** / **ـِيْنَ**; fem. **ة** → **ت** in dual, **ـَات** for sound fem. plural) and **جَمْعٌ مُكَسَّرٌ** (**جَمْعُ التَّكْسِيرِ** — broken plural; dual still on the singular stem; plural patterns learned by vocabulary). Includes exercise table (مُسْلِمٌ … مُؤْمِنٌ), Qurʾānic examples, and broken-plural vocabulary lists (e.g. **كِتَابٌ** → **كُتُبٌ**, **عَيْنٌ** → **أَعْيُنٌ** / **عُيُونٌ**, people / place / body-part plurals).
